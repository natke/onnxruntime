3617 /* \brief Set affinities for intra op threads
3618 *
3619 * Affinity string follows format:
3620 * logical_processor_id,logical_processor_id;logical_processor_id,logical_processor_id
3621 * Semicolon isolates configurations among threads, while comma split processors where ith thread expected to attach to.
3622 * e.g. 1,2,3;4,5
3623 * specifies affinities for two threads, with the 1st thread attach to the 1st, 2nd, and 3rd processor, and 2nd thread to the 4th and 5th.
3624 * To ease the configuration, an "interval" is also allowed:
3625 * e.g. 1-8;8-16;17-24
3626 * orders that the 1st thread runs on first eight processors, 2nd thread runs on next eight processors, and so forth.
3627 * Note:
3628 * 1. Once set, the number of thread affinities must equal to intra_op_num_threads - 1,
3629 * ort does not set affinity on the main thread which is started and managed by the calling app;
3630 * 2. For windows, ort will infer the group id from a logical processor id, for example, assuming there are two groups with each has 64 logical processors,
3631 * an id of 64 will be inferred as the last processor of the 1st group, while 65 will be interpreted as the 1st processor of the second group.
3632 * Hence 64-65 is an invalid configuration, because a windows thread cannot be attached to processors across group boundary.
3633 *
3634 * \since Version 1.14
3635 */
